Nginx负载均衡

Posted on 2018-10-23

Nginx(反向代理服务器)

一、正向代理

  1. 场景：在国内是无法正常使用google.com。如果想要访问google.com，可以购买一台国外的服务器A，此时你和服务器A的网络是相通的。而服务器A又跟google.com相通， 此时可以由服务器A代理你（客户端），去访问google.com。这个过程称之为正向代理，服务端（google.com）只需要知道代理服务器的ip，不需要知道客户端的ip。
  2. 结论：正向代理，是用于代理客户端的。

二、反向代理

  1. 场景：当一个服务器接受过多来自客户端的请求时，服务器难以处理和响应这些请求，会使得整个系统性能下降。为了解决这个难题，可以提供多台部署相同应用的服务器，让客户端的请求分别发送到不同的服务器上，这样单机服务器的压力就会降低很多，整体性能便会提升。但是有一个问题，每个服务器ip都是不同的，也就是说客户端的请求要发送到多个不同的ip上。让客户手动指定ip进行请求，这种方式很不明智。首先是客户的随机性，不知道会访问哪台服务器，其次，会造成一部分服务器压力大，一部分服务器几乎没有使用，浪费资源。因此，这里就需要一个角色去代理服务器，让客户端的请求直接发送到这个角色上，由这个角色去分发请求到不同的服务器上。
    这个角色就是反向代理服务器。
  2. 结论：反向代理，是用于代理服务端的。

三、 负载均衡

  1. 场景：反向代理过程中，每台服务器处理来自客户端的请求都应该是均衡的。
  2. 原理：使用一个反向代理服务器指向多台部署相同应用的服务器，客户端请求直接向反向代理服务器发起，反向代理服务器根据负载均衡机制，将请求转发到不同的应用服务器上。
  3. 负载均衡机制：
    nginx提供了一下三种方法：
    1）round-robin：请求以循环、轮转的方式分发到服务器
    2）least-connected：下一个请求被分配到拥有最少活动连接数的服务器
    3）ip-hash：使用一个哈希函数，基于客户端ip地址判断下一个请求应该被分发到哪台服务器

a）循环、轮转负载均衡
round-robin：默认情况下，使用循环、轮转的方式分发请求到服务器
配置示例：
http{
upstream myapp{
server srv1.example.com;
server srv2.example.com;
server srv3.example.com;
}
server{
listen 80;
location / {
proxy_pass http://myapp;
}
}
}

当不指定负载均衡方式时，默认以round-robin方式实现。所有请求都会被代理到myapp服务器，根据负载均衡机制分发请求。

b）最少连接负载均衡
least-connected：当一些请求处理的时间比较长时，最少连接负载均衡能够争取到更大的公平。
配置示例：
upstream myapp{
least-conn;
server srv1.example.com;
server srv2.example.com;
server srv3.example.com;
}

c）基于ip地址的负载均衡
ip-hash：采用目标地址散列调度（Destination Hashing Scheduling）算法，根据请求的目标ip地址，作为散列键（Hash Key）从静态分配的散列表找出对应的服务器，若该服务器可用且未超载，则将请求发送带服务器，否则返回空。
upstream myapp{
ip-hash;
server srv1.example.com;
server srv2.example.com;
server srv3.example.com;
}

Nginx和Zookeeper协同使用场景：https://blog.csdn.net/xiangjai/article/details/56844400
1.单纯使用Nginx，会造成配置维护成本变高。
2.单点故障率增加，因为热点服务的访问量很高，如果这个服务的负载均衡服务出现问题，整个服务都会挂点

Hello World~ Thank you for Read My Github Blog.
大家好，这是我的第一个博客。旧闻github社区是大老爷们的宅区，没事打打酱油，看下技术贴，了解下github社区的文化。
起初对个人博客不太感冒，原因有一下几点：

  1. 需要经常更新博客，发表新的内容。
  2. 维护个人博客，需要投入时间和精力。
  3. 对有需要解决的技术难点，通常看别人的博文就可以解决，无需自己动手。

以上内容可以总结为一个字：“懒”。

也是最近几个月才开始意识到这个问题，对于毕业不久的IT小白而言，未来发展可谓是浩瀚大海，不多努力，多挣扎一下怎么行呢？
稍微观察一下就可以发现，公司大概分为三种人：

  1. 工作热情低、工作推脱，办事拖拉
  2. 工作热情中、安于现状
  3. 工作热情高、积极进取
    第一类人比较少，我见过个别态度不认真的被劝退过，真的有！
    第二类人应该占70%~80%的比例吧，毕竟人是有惰性的，很难长时间保持高度热情。
    第三类人通常能够很好地定位自己，知道要做什么，学什么，以及下一步需要什么。
    毫无疑问，第三种人才有机会去吸收更多的知识，有可能是技术上的，也有可能是业务上的。
    你需要做也很简单，进行攀比。通过你与第一、二类人比较，你就会知道，安于现状对你目前的发展极为不利。
    以后总需要买房吧？出个远门需要有个交通工具吧？养儿育女是不是应该在才艺上也做下投资？
    这些都涉及到一个关键的点->Money！

所以，少年啊，好好奋斗吧！

  1. 总结生活经验和技术经验，便于巩固知识。
  2. 分享是一种美德。记录自己走过的坑，让猿友少踩雷区。
  3. 打响自己在业界的声望（技术再强，也要有人认可你才行啊）
